  - KPI Cards:
      * Total Patients (all time)
      * Today's Appointments
      * Active OPD Patients

      * Conversions (CRM Lead → MedID Patient)

      Monthly appointments, operations, OPD visits (last 6 months)

      Donut — New Patient, Follow-up, Emergency, Post-Op, Therapy

      Live list of today's scheduled appointments with doctor and time

      Pie chart — Instagram, Facebook, Google Ads, WhatsApp, YouTube breakdown

      Bar chart — avg days from inquiry to appointment, appointment to consultation

      Bar chart — how many CRM leads got converted to hospital patients (MedID)

  - Patient count, revenue, conversions → HMIS API

SECTION 2: ALL DATA SOURCES — MAPPED BY ORIGIN
================================================================================

-----------------------------------------------------------------------
2.1 DATA FROM HMIS (Hospital Management Information System)
-----------------------------------------------------------------------
  - Patient MedID (unique hospital patient ID)
  - Patient demographics (name, DOB, gender, blood type, address)
  - Patient medical history (conditions, diagnoses, medications)
  - Appointment records (all booked, completed, cancelled)
  - OPD queue and status
  - Operation Theatre schedule (surgery, surgeon, OR room, status)
  - Admission / discharge records
  - Doctor roster and availability
  - Billing / revenue data

INTEGRATION METHOD:
  - HMIS exposes REST API or HL7 FHIR endpoints (on-prem)
  - CRM backend polls or receives webhooks from HMIS
  - Bidirectional: CRM can also push appointments to HMIS

-----------------------------------------------------------------------
2.2 DATA FROM VOICE CALL AGENT (AI Calling Platform)
-----------------------------------------------------------------------
  - Call logs (date, time, duration, type: inbound/outbound/missed)
  - Call recording audio files
  - Full conversation transcripts
  - Caller identification (phone → mapped to CRM ID / MedID)
  - Call outcome (appointment booked, query resolved, etc.)
  - AI-generated sentiment score (positive/neutral/negative)
  - AI-generated action items from call
  - Lead capture from inbound inquiry calls
  - Appointment bookings made during call

INTEGRATION METHOD:
  - Voice Call Agent pushes call data to CRM via webhook on call completion
  - Audio files stored on-prem (shared file server or object storage)
  - Transcript and analysis pushed as JSON payload
  - CRM matches caller phone to patient/lead record

-----------------------------------------------------------------------
2.3 DATA FROM MARKETING AGENT (Digital Marketing AI Platform)
-----------------------------------------------------------------------
  - Lead form submissions (from QR code scans or form links)
  - Lead details: name, phone, email, city, inquiry, source
  - Campaign attribution (which campaign, which channel)
  - UTM parameters for source tracking
  - Funnel metrics: impressions, clicks, scans, form fills
  - Traffic by channel breakdown
  - Campaign performance stats

INTEGRATION METHOD:
  - Marketing Agent sends lead data to CRM via REST API POST on form submission
  - Campaign funnel metrics pushed periodically or on-demand
  - CRM stores all leads and maps them to campaigns

-----------------------------------------------------------------------
2.4 DATA FROM N8N (Workflow Automation)
-----------------------------------------------------------------------
  - Social media appointment requests (Instagram DMs, Facebook messages)
  - WhatsApp inquiries auto-parsed and logged
  - Website form submissions routed to CRM
  - Automated follow-up triggers based on CRM lead status

INTEGRATION METHOD:
  - n8n runs on-prem, connects to social media APIs
  - n8n webhooks post to CRM API when a booking/inquiry is captured
  - CRM creates lead or appointment record automatically
  - CRM can trigger n8n workflows (e.g., send WhatsApp follow-up)

SECTION 3: WHAT IS EXPECTED FROM AI AGENTS
================================================================================

-----------------------------------------------------------------------
3.1 VOICE CALL AI AGENT (MedVoice AI)
-----------------------------------------------------------------------
Expected Capabilities:
  - Answer inbound patient calls 24/7
  - Collect patient details (name, phone, inquiry, symptoms)
  - Check appointment availability (via CRM API)
  - Book, reschedule, or cancel appointments
  - Detect complaint severity and escalate to human CS agent
  - Generate post-call summary with action items
  - Transcribe full call in real-time or post-call
  - Score call sentiment (positive/neutral/negative)
  - Identify returning patients by phone number (via CRM/HMIS lookup)

Expected API Calls (Agent → CRM):
  POST /api/leads/call          → Create call lead
  POST /api/appointments/book   → Book appointment
  GET  /api/patients?phone=     → Look up existing patient
  POST /api/calls/log           → Submit call log + transcript
  POST /api/calls/recording     → Upload audio file reference

Data Payload on Call Completion:
  {
    callId, patientPhone, patientName, callType,
    duration, outcome, sentiment, transcript[],
    actionItems[], recordingUrl, agentType,
    appointmentBooked: { date, time, doctor, department }
  }

-----------------------------------------------------------------------
3.2 MARKETING AI AGENT
-----------------------------------------------------------------------
Expected Capabilities:
  - Serve lead capture forms via QR code and shareable link
  - Validate and sanitize submitted form data
  - Attribute lead to correct campaign and channel (UTM)
  - Track QR code scans and form open events
  - Push verified lead to CRM in real-time
  - Report campaign funnel metrics to CRM periodically
  - Score lead urgency (hot/warm/cold) based on inquiry keywords

Expected API Calls (Agent → CRM):
  POST /api/leads/marketing     → Submit new marketing lead
  POST /api/campaigns/:id/stats → Push funnel metrics update
  GET  /api/campaigns/:id       → Fetch campaign config for form display

Data Payload on Lead Submission:
  {
    campaignId, campaignName, source, utmParams,
    leadData: { name, phone, email, city, inquiry, formFields{} },
    captureMethod: "qr"|"form",
    timestamp, leadScore: "hot"|"warm"|"cold"
  }

-----------------------------------------------------------------------
3.3 CRM INTERNAL AI / ANALYTICS ENGINE
-----------------------------------------------------------------------
Expected Capabilities:
  - Compute CRM → MedID conversion rates across time periods
  - Calculate average lead turnaround time (inquiry to appointment)
  - Score lead hotness when not provided by agents
  - Detect duplicate leads/patients by phone / name matching
  - Flag overdue follow-ups automatically
  - Generate campaign performance reports on demand

SECTION 6: ON-PREMISES COMMUNICATION ARCHITECTURE
================================================================================

All services run on the hospital's internal server infrastructure.
NO patient data leaves the hospital network.

-----------------------------------------------------------------------
6.1 NETWORK TOPOLOGY
-----------------------------------------------------------------------

  [ Hospital LAN / VPN ]
       |
       |—— CRM Frontend     (React, served by Nginx)         PORT 3000
       |—— CRM Backend API  (Node.js / Express)              PORT 5000
       |—— PostgreSQL DB    (on-prem database server)         PORT 5432
       |—— HMIS System      (existing hospital software)      PORT varies
       |—— Voice Call Agent (AI telephony platform)           PORT varies
       |—— Marketing Agent  (form / campaign engine)          PORT varies
       |—— n8n              (workflow automation, on-prem)    PORT 5678
       |—— Redis Cache      (session + queue management)      PORT 6379
       |—— File Storage     (call recordings, QR assets)      NFS / MinIO

-----------------------------------------------------------------------
6.2 COMMUNICATION FLOWS
-----------------------------------------------------------------------

FLOW 1: Patient calls hospital
  Phone → Voice Call Agent AI
       → AI looks up patient (GET /api/patients?phone=)
       → AI books appointment if needed (POST /api/appointments/book)
       → On call end: AI pushes full call log (POST /webhooks/voice-agent)
       → CRM stores transcript, recording ref, lead/log record
       → CRM syncs booked appointment to HMIS (POST HMIS API)

FLOW 2: Marketing lead submits form (QR / link)
  Patient scans QR / opens link
       → Marketing Agent serves form
       → Patient submits form
       → Marketing Agent scores lead (hot/warm/cold)
       → Marketing Agent POSTs to CRM (POST /webhooks/marketing)
       → CRM stores lead under campaign
       → CRM staff follows up

FLOW 3: Social media inquiry / booking via n8n
  Patient messages on Instagram / Facebook / WhatsApp
       → n8n detects message via social API
       → n8n parses inquiry, creates structured payload
       → n8n POSTs to CRM (POST /webhooks/n8n)
       → CRM creates lead or appointment
       → CRM (optionally) triggers n8n to send confirmation reply

FLOW 4: HMIS patient registration (new patient gets MedID)
  Patient registers at hospital counter
       → HMIS assigns MedID
       → HMIS pushes patient data to CRM (POST /webhooks/hmis)
           OR CRM polls HMIS periodically (GET HMIS /patients/new)
       → CRM matches by phone to existing CRM lead
       → CRM marks lead as "Converted", stores MedID

FLOW 5: HMIS appointment sync
  Appointment booked from HMIS (reception desk)
       → HMIS pushes to CRM webhook (POST /webhooks/hmis/appointments)
       → CRM stores appointment, links to patient record
       → Dashboard and calendar reflect new appointment

FLOW 6: CRM books appointment → HMIS sync
  CRM staff or voice agent books appointment in CRM
       → CRM calls HMIS API to create appointment in HMIS
       → HMIS returns appointment ID
       → CRM stores HMIS appointment ID for reference

-----------------------------------------------------------------------
6.3 INTERNAL SERVICE COMMUNICATION
-----------------------------------------------------------------------

  CRM Backend ←→ PostgreSQL     : Direct TCP (port 5432), pooled connections
  CRM Backend ←→ Redis          : Session cache, job queues (port 6379)
  CRM Backend ←→ HMIS           : REST API over internal LAN (HTTPS/HTTP)
  CRM Backend ←→ Voice Agent    : Webhook receiver + REST calls to agent API
  CRM Backend ←→ Marketing Agent: Webhook receiver + REST calls to agent API
  CRM Backend ←→ n8n            : Webhook receiver (n8n POSTs to CRM)
                                   CRM can also POST to n8n trigger URL
  CRM Frontend ←→ CRM Backend   : REST API (HTTPS internal), JWT auth
  File Storage ←→ CRM Backend   : MinIO S3-compatible API or NFS mount

-----------------------------------------------------------------------
6.4 SECURITY CONSIDERATIONS (On-Prem)
-----------------------------------------------------------------------

  - All internal service communication over HTTPS (self-signed cert or internal CA)
  - JWT tokens with short expiry (15 min access, 7 day refresh)
  - Webhook endpoints secured with shared secret headers (X-Webhook-Secret)
  - PostgreSQL: dedicated CRM user with least-privilege access
  - Redis: password protected, no public access
  - File storage (recordings): access only via signed URLs, not public
  - Audit log table for all data mutations (who changed what, when)
  - Role-based access control (admin, doctor, reception, marketing roles)
  - No external internet required for core functionality
  - Optional: VPN for remote staff access to CRM

-----------------------------------------------------------------------
6.5 BACKGROUND JOBS (Queue-based via Redis + Bull/BullMQ)
-----------------------------------------------------------------------

  - HMIS Sync Job         : Every 5 min — pull new patients, appointments
  - Follow-up Reminder    : Daily — flag leads with overdue follow-up dates
  - Conversion Matcher    : On new HMIS patient → match to CRM leads by phone
  - Campaign Stats Rollup : Hourly — aggregate marketing funnel metrics
  - Call Log Processor    : On webhook receive — process transcript, score lead
  - Report Generator      : On demand — generate campaign PDF report
